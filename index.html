Maze bounce set

The idea

The idea was, what happens when you bounce a ray arround a maze, how many times does it bounce before it comes back, what depends on this number of bounces. So I decided to visualise this.

This visualisation shows, how many times a ray needs to bounce back from every frame if fired from a certain angle.

The algorithm

parameters:

  • angle - the angle at which the ray will be from
  • resolution - how many times the canvas will be divided
  1. Choose a starting point
  2. Make a ray (linear function) at the requested angle
  3. Bounce the ray from the first wall
  4. Repeat untill ray is back at the start (untill the ray is close to the starting point)
